# Execution Plan: Performance Improvements

## 1. 문서 목적과 범위 [DOC-1]
- 목적: 본 문서는 구현 단계에서 설계 변경 없이 그대로 따라야 하는 실행 기준 문서다. [DOC-1.1]
- 범위: Next.js 15 App Router 기반 대시보드 전반의 렌더링 방식, 캐싱, 클라이언트 fetch 제거에 대한 합의된 개선 사항만 포함한다. [DOC-1.2]

## 2. 전체 작업 원칙 (변경 금지 사항 포함) [RULE-2]
- 해야 할 것 [RULE-2.1]
  - 이 문서의 체크리스트를 그대로 이행한다. [RULE-2.1.1]
  - 코드 변경 전후에 빌드/린트/타입체크를 수행한다. [RULE-2.1.2]
  - 캐싱과 revalidation은 명시된 키/태그 전략을 준수한다. [RULE-2.1.3]
- 하지 말 것 [RULE-2.2]
  - 본 문서에 없는 새로운 최적화 아이디어를 추가하지 않는다. [RULE-2.2.1]
  - 기존 설계 의도를 변경하거나 재해석하지 않는다. [RULE-2.2.2]
  - 강제적으로 동작 방식을 바꾸는 리팩터링을 포함하지 않는다. [RULE-2.2.3]

## 3. 작업 범위 요약 [SCOPE-3]
- force-dynamic 제거 대상 페이지/레이아웃 정리 및 제거 [SCOPE-3.1]
- unstable_cache 적용 대상 서버 함수에 캐싱/태그 설계 적용 [SCOPE-3.2]
- row/card 내부 client fetch 제거 및 서버 데이터 전달로 대체 [SCOPE-3.3]
- Optional 항목은 후순위로 분리하여 별도 작업으로 취급 [SCOPE-3.4]

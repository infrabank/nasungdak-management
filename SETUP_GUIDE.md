## 1. 계정 생성

### 관리자가 해야 할 일

신규 사용자 계정은 관리자가 생성합니다.

**필요 정보:**
- 이메일 주소
- 이름
- 초기 비밀번호

> ⚠️ **보안 주의**: 초기 비밀번호는 사용자에게 전달 후 반드시 변경하도록 안내하세요.

---

## 2. 조직 설정

### 2.1 조직(Organization) 생성

조직은 데이터 격리의 최상위 단위입니다. 각 프랜차이즈/사업체별로 하나의 조직을 생성합니다.

**조직 정보:**
- 조직명 (예: "나성닭강정")
- 슬러그 (예: "nasungdak-default")

### 2.2 조직 멤버십 할당

사용자를 조직에 연결합니다.

**역할 종류:**
| 역할 | 설명 |
|------|------|
| `owner` | 조직 소유자 (모든 권한) |
| `admin` | 관리자 (대부분의 권한) |
| `member` | 일반 멤버 (제한된 권한) |

> 💡 **중요**: 사용자가 조직에 소속되지 않으면 기초 데이터(메뉴, 재료, SKU 등)를 볼 수 없습니다.

---

## 3. 매장 설정

### 3.1 매장(Store) 생성

각 물리적 매장을 등록합니다.

**대시보드 > 매장 관리**에서:
1. "매장 추가" 클릭
2. 매장 정보 입력:
   - 매장명 (예: "나성닭강정 본점")
   - 주소 (선택)
   - 연락처 (선택)
3. 저장

### 3.2 사용자-매장 할당

사용자가 접근할 수 있는 매장을 지정합니다.

> ⚠️ **중요**: 사용자가 매장에 할당되지 않으면 매입/판매 데이터를 볼 수 없습니다.

**할당 방법:**
1. 사용자 관리 페이지 접속
2. 해당 사용자 선택
3. "매장 할당" 에서 접근 가능한 매장 선택
4. 역할 지정 (store_owner, manager, staff 등)

## 4. 기초 데이터 등록

기초 데이터는 매입/판매 등록의 기반이 됩니다. **반드시 순서대로** 등록하세요.

### 4.1 메뉴 카테고리 등록

**대시보드 > 기초데이터 > 메뉴**

메뉴는 상위 분류입니다.

**예시:**
| 메뉴명 | 설명 |
|--------|------|
| 닭강정 | 메인 메뉴 |
| 떡볶이 | 사이드 메뉴 |
| 음료 | 음료류 |
| 분식 | 기타 분식 |

**CSV 일괄 등록 가능** (메뉴명, 설명, 활성 컬럼)

### 4.2 재료 등록

**대시보드 > 기초데이터 > 재료**

매입에 사용되는 원재료입니다.

**예시:**
| 재료명 | 단위 |
|--------|------|
| 닭고기 | kg |
| 소스 | L |
| 튀김유 | L |
| 떡 | kg |

**CSV 일괄 등록 가능**

### 4.3 SKU 등록

**대시보드 > 기초데이터 > SKU**

SKU(Stock Keeping Unit)는 판매 단위입니다.

**예시:**
| SKU명 | 메뉴 | 단가 |
|-------|------|------|
| 닭강정 소 | 닭강정 | 15,000원 |
| 닭강정 중 | 닭강정 | 20,000원 |
| 닭강정 대 | 닭강정 | 25,000원 |
| 떡볶이 1인분 | 떡볶이 | 5,000원 |

> 💡 SKU의 단가는 판매 금액 계산에 사용됩니다.

**CSV 일괄 등록 가능**

### 4.4 공급업체 등록 (선택)

**대시보드 > 기초데이터 > 공급업체**

매입처 정보입니다.

**예시:**
| 업체명 | 연락처 | 품목 |
|--------|--------|------|
| OO식품 | 010-1234-5678 | 닭고기 |
| XX유통 | 010-9876-5432 | 소스, 튀김유 |

### 4.5 메뉴-재료 매핑 (선택)

**대시보드 > 기초데이터 > 메뉴-재료**

메뉴와 재료의 연결 관계입니다. 매입 검증에 사용됩니다.

**예시:**
| 메뉴 | 재료 | 기준량 |
|------|------|--------|
| 닭강정 | 닭고기 | 0.5kg |
| 닭강정 | 소스 | 0.1L |
| 떡볶이 | 떡 | 0.3kg |

> 이 매핑이 있으면 매입 등록 시 "메뉴-재료 조합이 올바른지" 자동 검증됩니다.

## 5. 운영 시작

### 5.1 일일 매입 등록

**대시보드 > 매입 관리**

1. "매입 등록" 클릭
2. 정보 입력:
   - 날짜
   - 메뉴 선택
   - 재료 선택
   - 공급업체
   - 수량, 단가
3. 저장

**CSV 일괄 업로드 지원**

### 5.2 일일 판매 등록

**대시보드 > 판매 관리**

1. "판매 등록" 클릭
2. 정보 입력:
   - 날짜
   - SKU 선택
   - 판매 수량
3. 저장

**CSV 일괄 업로드 지원**

### 5.3 대시보드 확인

**대시보드 > 홈**

- 일일/월간 매출 현황
- 매입 대비 매출 비교
- 주요 지표 확인

## 문제 해결

### 기초 데이터가 보이지 않음

**원인**: 사용자가 조직에 연결되지 않음

**해결**: 
1. `organization_members` 테이블에서 사용자-조직 연결 확인
2. 없으면 연결 추가
3. 로그아웃 후 다시 로그인

### 매입/판매 데이터가 보이지 않음

**원인**: 사용자가 매장에 할당되지 않음

**해결**:
1. `user_store_assignments` 테이블에서 사용자-매장 연결 확인
2. 없으면 연결 추가
3. 로그아웃 후 다시 로그인

### 새로 등록한 데이터가 목록에 안 보임

**원인**: 캐시

**해결**:
1. 페이지 강력 새로고침 (Ctrl+Shift+R)
2. 로그아웃 후 다시 로그인
3. 그래도 안 되면 관리자에게 캐시 무효화 요청

## 데이터 구조 요약

```
조직 (Organization)
  └── 기초 데이터
      ├── 메뉴 카테고리
      ├── 재료
      ├── SKU
      ├── 공급업체
      └── 메뉴-재료 매핑

매장 (Store)
  └── 거래 데이터
      ├── 매입 내역
      └── 판매 내역

사용자 (User)
  ├── 조직 멤버십 → 기초 데이터 접근
  └── 매장 할당 → 거래 데이터 접근
```

환영합니다! 이 가이드는 시스템을 처음 사용하시는 분들을 위한 단계별 설정 안내입니다.

## 📋 빠른 체크리스트

시작하기 전에, 아래 항목이 모두 완료되어야 시스템을 정상적으로 사용할 수 있습니다.

| 순서 | 항목           | 필수 | 담당          |
| :--: | -------------- | :--: | ------------- |
|  1   | 계정 생성      |  ✅  | 관리자        |
|  2   | 조직 연결      |  ✅  | 관리자        |
|  3   | 매장 생성      |  ✅  | 관리자/사용자 |
|  4   | 매장 권한 할당 |  ✅  | 관리자        |

|  7   | SKU 등록       |  ✅  | 사용자        |

## 🚀 STEP 1. 계정 & 권한 설정

> 🔐 이 단계는 **관리자**가 진행합니다.

### 1-1. 사용자 계정 생성

```
필요 정보
├── 📧 이메일 (로그인 ID로 사용)
├── 👤 이름
└── 🔑 초기 비밀번호
```

### 1-2. 조직(Organization) 연결

**조직이란?** 데이터를 구분하는 최상위 단위입니다.

```
예시 구조
🏢 나성닭강정 (조직)
    ├── 📦 기초 데이터 (메뉴, 재료, SKU...)
    ├── 🏪 본점
    └── 🏪 새롬점
```

| 역할     | 권한                       |
| -------- | -------------------------- |
| `owner`  | 모든 권한 (조직 설정 포함) |
| `admin`  | 대부분의 권한              |
| `member` | 기본 권한                  |

> ⚠️ **주의**: 조직에 연결되지 않으면 기초 데이터를 볼 수 없습니다!

### 1-3. 매장 권한 할당

사용자가 접근할 수 있는 매장을 지정합니다.

```
사용자 권한 예시
👤 홍길동
    ├── 🏪 본점 (manager)
    └── 🏪 새롬점 (staff)
```

> ⚠️ **주의**: 매장 권한이 없으면 매입/판매 데이터를 볼 수 없습니다!

---

## 🏪 STEP 2. 매장 설정

> 📍 **대시보드 → 매장 관리**

### 매장 추가하기

1. **"+ 매장 추가"** 버튼 클릭
2. 정보 입력:

| 항목   | 필수 | 예시             |
| ------ | :--: | ---------------- |
| 매장명 |  ✅  | 나성닭강정 본점  |
| 주소   |  ⬜  | 서울시 강남구... |
| 연락처 |  ⬜  | 02-1234-5678     |

3. **저장** 클릭

---

## 📝 STEP 3. 기초 데이터 등록

> ⚡ **중요**: 반드시 아래 순서대로 등록하세요!

```
등록 순서
1️⃣ 메뉴 → 2️⃣ 재료 → 3️⃣ SKU → 4️⃣ 공급업체 → 5️⃣ 메뉴-재료 매핑
```

---

### 3-1. 메뉴 카테고리 등록

> 📍 **대시보드 → 기초데이터 → 메뉴**

메뉴는 상품의 **상위 분류**입니다.

#### 예시 데이터

| 메뉴명 | 설명        | 활성 |
| ------ | ----------- | :--: |
| 닭강정 | 메인 메뉴   |  ✅  |
| 떡볶이 | 사이드 메뉴 |  ✅  |
| 음료   | 음료류      |  ✅  |
| 분식   | 기타 분식   |  ✅  |

#### 등록 방법

- **개별 등록**: "+ 메뉴 추가" 버튼
- **일괄 등록**: "CSV 업로드" 버튼

<details>
<summary>📄 <b>CSV 형식 보기</b></summary>

```csv
메뉴명,설명,활성
닭강정,메인 메뉴,true
떡볶이,사이드 메뉴,true
음료,음료류,true
```

</details>

---

### 3-2. 재료 등록

> 📍 **대시보드 → 기초데이터 → 재료**

매입에 사용되는 **원재료** 목록입니다.

#### 예시 데이터

| 재료명 | 단위 |
| ------ | ---- |
| 닭고기 | kg   |
| 소스   | L    |
| 튀김유 | L    |
| 떡     | kg   |
| 어묵   | kg   |

<details>
<summary>📄 <b>CSV 형식 보기</b></summary>

```csv
재료명,단위
닭고기,kg
소스,L
튀김유,L
떡,kg
```

</details>

---

### 3-3. SKU 등록

> 📍 **대시보드 → 기초데이터 → SKU**

**SKU란?** 실제 판매되는 상품 단위입니다.

#### 예시 데이터

| SKU명        | 메뉴   |     단가 |
| ------------ | ------ | -------: |
| 닭강정 소    | 닭강정 | 15,000원 |
| 닭강정 중    | 닭강정 | 20,000원 |
| 닭강정 대    | 닭강정 | 25,000원 |
| 떡볶이 1인분 | 떡볶이 |  5,000원 |
| 콜라 500ml   | 음료   |  2,000원 |

> 💡 SKU의 **단가**는 판매 금액 자동 계산에 사용됩니다.

<details>
<summary>📄 <b>CSV 형식 보기</b></summary>

```csv
SKU명,메뉴,단가
닭강정 소,닭강정,15000
닭강정 중,닭강정,20000
닭강정 대,닭강정,25000
```

</details>

---

### 3-4. 공급업체 등록 (선택)

> 📍 **대시보드 → 기초데이터 → 공급업체**

매입처 정보입니다.

| 업체명 | 연락처        | 주요 품목    |
| ------ | ------------- | ------------ |
| OO식품 | 010-1234-5678 | 닭고기       |
| XX유통 | 010-9876-5432 | 소스, 튀김유 |

---

### 3-5. 메뉴-재료 매핑 (선택)

> 📍 **대시보드 → 기초데이터 → 메뉴-재료**

메뉴와 재료의 연결 관계를 설정하면 **매입 자동 검증**이 가능합니다.

| 메뉴   | 재료   | 기준량 |
| ------ | ------ | ------ |
| 닭강정 | 닭고기 | 0.5kg  |
| 닭강정 | 소스   | 0.1L   |
| 떡볶이 | 떡     | 0.3kg  |
| 떡볶이 | 어묵   | 0.1kg  |

> 💡 매핑 설정 시, 매입 등록할 때 "메뉴-재료 조합이 맞는지" 자동으로 확인됩니다.

---

## 💼 STEP 4. 운영 시작

기초 데이터 등록이 완료되면 일일 운영을 시작할 수 있습니다.

### 4-1. 매입 등록

> 📍 **대시보드 → 매입 관리**

**일일 원재료 매입 내역**을 기록합니다.

```
입력 항목
├── 📅 날짜
├── 🍗 메뉴 선택
├── 📦 재료 선택
├── 🏢 공급업체
├── 📊 수량
└── 💰 단가
```

#### 등록 방법

- **개별 등록**: "+ 매입 등록" 버튼
- **일괄 등록**: "CSV 업로드" 버튼

---

### 4-2. 판매 등록

> 📍 **대시보드 → 판매 관리**

**일일 판매 내역**을 기록합니다.

```
입력 항목
├── 📅 날짜
├── 🏷️ SKU 선택
└── 📊 판매 수량
```

> 💡 판매 금액은 **SKU 단가 × 수량**으로 자동 계산됩니다.

---

### 4-3. 대시보드 확인

> 📍 **대시보드 → 홈**

등록된 데이터를 바탕으로 다양한 현황을 확인할 수 있습니다.

- 📈 일일/월간 매출 현황
- 📊 매입 대비 매출 비교
- 🎯 주요 지표

## ❓ 자주 묻는 질문 (FAQ)

<details>
<summary><b>Q. 기초 데이터가 보이지 않아요</b></summary>

**원인**: 조직(Organization)에 연결되지 않음

**해결 방법**:

1. 관리자에게 조직 연결 요청
2. 로그아웃 후 다시 로그인

</details>

<details>
<summary><b>Q. 매입/판매 데이터가 보이지 않아요</b></summary>

**원인**: 매장 권한이 할당되지 않음

**해결 방법**:

1. 관리자에게 매장 권한 요청
2. 로그아웃 후 다시 로그인

</details>

<details>
<summary><b>Q. 새로 등록한 데이터가 목록에 안 보여요</b></summary>

**원인**: 브라우저 캐시

**해결 방법**:

1. 페이지 새로고침 (F5)
2. 강력 새로고침 (Ctrl + Shift + R)
3. 로그아웃 후 다시 로그인

</details>

<details>
<summary><b>Q. 로그인이 안 돼요</b></summary>

**가능한 원인**:

- 비밀번호 오류 (여러 번 실패 시 일시 차단)
- 계정 비활성화

**해결 방법**:

1. 5분 후 다시 시도
2. 관리자에게 비밀번호 초기화 요청

</details>

